# TMI Climate Solutions — AI Quoting Assistant

A RAG-powered quoting assistant for TMI Climate Solutions. Engineers fill in a short request for quotation form and the system retrieves the most similar historical quotes, applies distilled engineering rules from past projects, and streams a fully structured draft quote in real time. Each section can be edited, regenerated, and approved independently before export.

---

## What it does

- Matches a new RFQ against 30 historical quote documents using semantic search
- Retrieves engineering rules from a knowledge base built from all past projects
- Streams a complete 7-section quote draft using Claude as the generation model
- Lets engineers review, edit, regenerate, and approve each section independently
- Assigns each section to an SME role (Engineering Lead, Sales Director, etc.) and tracks approvals
- Runs four mechanical checks (section completeness, location reference, pricing formula, price comparison) automatically
- Runs an LLM judge scoring the draft on five dimensions with a final APPROVE / REVISE / REJECT verdict
- Logs every engineer correction for future knowledge base improvement

## Project structure

```
quotes/               30 historical quote PDFs (source documents)
extracted/            Extracted text files, one per PDF (generated by pipeline)
pipeline/
  extract.py          Step 1 — PDF to text extraction
  embed.py            Step 2 — Embed documents into ChromaDB
  extract_rules.py    Step 3 — Extract and embed engineering rules
  retrieve.py         Shared query library for both ChromaDB collections
  eval.py             Offline CLI evaluation harness
api/
  main.py             FastAPI backend — all endpoints
frontend/
  index.html          Entire UI — single HTML file, no build step
chroma_store/         ChromaDB persistent store (generated, not committed)
engineering_rules.json  356 extracted engineering rules (generated by pipeline)
corrections.json      Append-only log of engineer corrections (created at runtime)
assignments.json      Append-only log of SME assignments (created at runtime)
claude.md             Coding standards and project constraints
```

## Setup

### Prerequisites

- Python 3.9+
- An Anthropic API key

### Install dependencies

```bash
pip install fastapi uvicorn anthropic python-dotenv chromadb sentence-transformers pdfplumber requests
```

### Add your API key

Create a `.env` file in the project root:

```
ANTHROPIC_API_KEY=your_key_here
```

### Run the pipeline (first time only)

These three steps populate the vector database. Run them once from the project root. You only need to re-run them if you add new quote documents.

```bash
# Step 1 — extract text from all PDFs
python pipeline/extract.py

# Step 2 — embed documents into ChromaDB
python pipeline/embed.py

# Step 3 — extract and embed engineering rules
python pipeline/extract_rules.py
```

After these complete you will have:
- `extracted/` — 30 `.txt` files
- `chroma_store/` — ChromaDB with two collections: `tmi_quotes` (30 documents) and `engineering_rules` (356 rules)
- `engineering_rules.json` — human-readable rule list

### Start the API server

```bash
python -m uvicorn api.main:app --reload
```

The server runs on `http://localhost:8000`.

### Open the frontend

Open `frontend/index.html` directly in your browser. No server or build step needed.

## Using the system

1. Fill in the six RFQ fields — facility size, cooling load, configuration type, location, redundancy tier, timeline. Each field has autocomplete suggestions.
2. Click **Generate Quote**. The system retrieves the three closest historical quotes, fetches relevant engineering rules, and streams the draft.
3. Review the **Matched Historical Quotes**. Click any card to see the pricing summary and engineering notes from that document. Use **View PDF** to open the original.
4. Expand **Engineering Rules Applied** to see which rules shaped the draft. Edit, remove, or add rules before regenerating.
5. Work through each section of the quote. Use **Edit** to modify text directly, **Re-generate** to stream a new version of just that section using the current rules, and **Approve** to lock the section.
6. Click the **assignee chip** on any section header to reassign it to a different SME role. The progress bar tracks both section approvals and SME review coverage.
7. Check the **AI Evaluation Report** bar for mechanical check results and LLM judge scores.
8. Once all sections are approved, the **Export to CPQ** button appears.

## API endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/retrieve` | Vector search — returns top 3 matching historical quotes |
| POST | `/generate-quote-stream` | Streams the full quote as SSE |
| POST | `/regenerate-section` | Streams a single section regeneration as SSE |
| POST | `/run-eval` | Runs four mechanical checks on the generated quote |
| POST | `/run-judge` | LLM judge — returns five dimension scores and a verdict |
| POST | `/log-correction` | Appends an engineer correction to corrections.json |
| POST | `/assign-section` | Appends an SME assignment to assignments.json |
| POST | `/retrieve-rules` | Query the engineering rules knowledge base directly |
| GET  | `/pdf/{filename}` | Serves a historical quote PDF for inline viewing |

## Pricing formula

All quotes follow TMI's standard formula:

```
Overhead      = (Materials + Labour) × 18%
Pre-margin    = Materials + Labour + Overhead
Margin        = Pre-margin × 22/78
Total quoted  = Pre-margin / 0.78
```

The mechanical eval check verifies this formula is correct in every generated quote.

## Models used

- **Generation and judging** — `claude-sonnet-4-6` via the Anthropic API
- **Embeddings** — `all-MiniLM-L6-v2` via sentence-transformers (runs locally, no API cost)
- **Vector store** — ChromaDB, persistent on disk

## What is not yet built

- **Corrections feedback loop** — `corrections.json` is populated but not yet fed back into the knowledge base. A nightly job that re-runs rule extraction on corrections and re-embeds them would close this loop.
- **Session persistence** — refreshing the page loses all in-progress work. localStorage serialisation or a backend session store would fix this.
- **Authentication** — the API has no auth. All endpoints are open on localhost.
- **SME notifications** — assignments are logged but assigned SMEs are not notified.
- **Chunked deduplication** — the rule deduplication step in `extract_rules.py` fails silently when the full rule list is too large for a single Claude call. A chunked approach processing ~50 rules at a time would improve knowledge base quality.
